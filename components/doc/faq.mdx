# Frequently Asked Questions

Below are some common questions about Vulhub and their answers.

## General Questions

### What is Vulhub?

Vulhub is an open-source collection of pre-built vulnerable docker environments. It's designed to help security researchers, educators, and students learn about and practice exploiting various security vulnerabilities in a safe, controlled environment.

### Is Vulhub free to use?

Yes, Vulhub is completely free and open-source. You can use it for personal learning, educational purposes, or in a professional setting.

### Can I contribute to Vulhub?

Absolutely! Vulhub welcomes contributions from the community. See our [Contributing Guide](/documentation/contributing) for more information.

## Technical Questions

### Do I need to know Docker to use Vulhub?

Basic knowledge of Docker and Docker Compose is helpful, but not strictly necessary. The documentation provides step-by-step instructions for each vulnerability environment.

### Are the vulnerabilities in Vulhub real?

Yes, all vulnerabilities in Vulhub are based on real-world CVEs and security issues that have affected actual software.

### Is it safe to run Vulhub on my production server?

No, you should never run Vulhub on a production server. The environments are intentionally vulnerable and should only be run in isolated, controlled environments for educational purposes.

### How do I update Vulhub?

You can update your local copy of Vulhub by pulling the latest changes from the GitHub repository:

```bash
git pull origin master
```

## Troubleshooting

### I'm having issues with Docker Compose

Make sure you have the latest version of Docker and Docker Compose installed. If you're still having issues, check the [GitHub Issues](https://github.com/vulhub/vulhub/issues) page to see if others have encountered similar problems.

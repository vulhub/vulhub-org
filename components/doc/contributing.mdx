# Contributing to Vulhub

We welcome contributions to Vulhub! This guide will help you get started with contributing to the project.

## Types of Contributions

There are several ways you can contribute to Vulhub:

1. **Adding new vulnerable environments**
2. **Improving existing environments**
3. **Enhancing documentation**
4. **Reporting bugs**

## Guidelines for Adding New Environments

When adding a new vulnerable environment, please follow these guidelines:

- Each environment should be in its own directory
- Include a detailed README.md with:
  - Description of the vulnerability
  - Steps to reproduce
  - References to CVEs or other sources
- Make sure the environment is reproducible using Docker Compose

## Pull Request Process

1. Fork the repository
2. Create a new branch for your changes
3. Make your changes
4. Test your environment thoroughly
5. Submit a pull request

## Code of Conduct

Please follow our code of conduct when contributing to the project. Be respectful and considerate of other contributors.

## Need Help?

If you need help with contributing, please open an issue on GitHub or reach out to the maintainers.
